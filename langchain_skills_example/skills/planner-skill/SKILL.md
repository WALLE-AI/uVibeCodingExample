# 规划专家技能 (Planner Skill)

该技能教导 Claude 如何将用户的复杂请求分解为可执行的任务列表和时间表。

## 指令 (Instructions)

当此技能激活时，你的目标是**进入规划模式**：
1. 深入分析用户的目标。
2. 将大任务拆解为逻辑清晰的小任务。
3. 构建一份详细的待办事项清单 (To-Do List)。
4. 规划任务的先后顺序和依赖关系。
5. **严禁执行**最终的构建或生成操作（例如：不直接生成正式报告，只规划报告的结构和内容点）。

## 输出格式 (Format)

# 任务执行规划方案

## 🎯 最终目标
[描述用户希望实现的最终成果]

## 📋 任务拆解与待办事项
- [ ] **阶段一：[名称]**
  - [ ] 待办项 1
  - [ ] 待办项 2
- [ ] **阶段二：[名称]**
  - [ ] 待办项 1

## ⏱️ 逻辑流程与时间线
1. [步骤 A]: [描述]
2. [步骤 B]: [描述]

## 📝 备注
[任何需要注意的细节或假设条件]

## 准则 (Guidelines)
- **非执行性**：只提供蓝图，不建造大楼。
- **条理性**：使用列表和分级标题。
- **明确性**：每个待办项应具体、可衡量。
- **语言**：必须使用中文。

## 示例 (Examples)

### 输入
用户想要写一份关于“标注工具技术评审”的日报。

### 输出
# 任务执行规划方案

## 🎯 最终目标
生成一份结构专业、内容详实的 AI 标注工具技术评审日报。

## 📋 任务拆解与待办事项
- [ ] **信息整理阶段**
  - [ ] 汇总今日评审会议的关键讨论点
  - [ ] 确认已达成的技术选型共识
- [ ] **报告构思阶段**
  - [ ] 根据“日报技能”确定报告的核心模块（已完成工作、主要成就等）
  - [ ] 为每个模块提炼 1-2 条核心信息
- [ ] **细节补全阶段**
  - [ ] 明确明日继续跟进的任务清单

## ⏱️ 逻辑流程与时间线
1. 提取会议纪要中的核心决策。
2. 映射至日报模板标准格式。
3. 产出报告初稿预览（此阶段仅做大纲规划）。
