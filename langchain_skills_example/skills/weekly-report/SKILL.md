# 每周报告技能 (Weekly Report Skill)

该技能教导 Claude 如何从多份日报中提炼核心价值，生成一份面向管理层的执行级周报。

## 指令 (Instructions)

当此技能激活时：
1. 汇总本周所有日报的内容。
2. 识别出本周最重要的里程碑和产出。
3. 评估潜在风险并制定下周计划。
4. 使用专业的中文商务语气进行表述。

## 报告格式 (Format)

# 每周执行摘要 - [日期范围]

## 🌟 总体概述
[用 2-3 句话对本周进展进行高水平总结，重点突出对项目的贡献]

## 📈 重大里程碑
- [里程碑 1]
- [里程碑 2]

## 📊 团队表现与指标
- 已完成工作项：[数量]
- 重点关注领域：[本周核心攻坚方向]

## 🔮 下周战略重点
- [目标 1]
- [目标 2]

## ⚠️ 风险评估
- [风险描述]：[对应的缓解计划或建议]

## 准则 (Guidelines)
- **语气**：执行级（Structured for executives），聚焦战略影响而非技术细节。
- **聚合性**：不要只是简单重复日报，要进行深度的归纳和分类。
- **前瞻性**：确保风险评估和下周计划具有指导意义。

## 示例 (Examples)

### 输入
周一：完成 API 联调。周二：修复生产环境 Bug。周三：编写性能测试报告。

### 输出
# 每周执行摘要 - 2026年1月26日 - 1月30日

## 🌟 总体概述
本周项目进入了关键的集成测试阶段。通过高效的 API 联调和及时的生产环境故障修复，确保了系统的稳定性，并为下周的 beta 测试打下了坚实基础。

## 📈 重大里程碑
- 核心系统集成：成功完成前后端 API 联调，实现端到端流程跑通。

## 📊 团队表现与指标
- 已完成工作项：8项
- 重点关注领域：系统稳定性与性能分析。

## 🔮 下周战略重点
- 启动全量 Beta 测试。
- 根据性能测试报告优化数据库查询。

## ⚠️ 风险评估
- **并发压力**：测试显示高负载下会有延迟。**缓解计划**：下周二前上线缓存优化方案。
