# 每日报告技能 (Daily Report Skill)

该技能教导 Claude 如何高效地将零散的活动记录整理成一份标准化的中文日报。

## 指令 (Instructions)

当此技能激活时，请遵循以下流程：
1. 分析用户提供的活动列表。
2. 提取关键成果、遇到的困难以及明日计划。
3. 按照指定的 Markdown 格式生成报告。

## 报告格式 (Format)

# 每日工作进展报告 - [日期]

## 📝 已完成的工作
- [工作项]：[简要描述]

## 🚀 主要成就
- [列出当日最显著的 1-2 个成就]

## 🚩 挑战与阻碍
- [问题描述]（状态：[已解决/进行中]）

## 📅 明日计划
- [任务 1]
- [任务 2]

## 准则 (Guidelines)
- **语言**：必须使用中文。
- **语气**：专业、客观且简洁。
- **重点**：突出价值和成果，而非仅仅罗列动作。
- **时间**：如果用户未提供日期，请使用当前日期。

## 示例 (Examples)

### 输入
- 修复了登录页面的 CSS 样式问题
- 参加了需求评审会议
- 完成了 API 文档的初稿

### 输出
# 每日工作进展报告 - 2026年1月30日

## 📝 已完成的工作
- 界面优化：修复了登录页面的 CSS 样式，解决了移动端显示异常的问题。
- 团队沟通：参与需求评审，明确了下周 V1.2 版本的开发重点。
- 文档编写：完成了核心 API 接口文档的初稿。

## 🚀 主要成就
- 成功交付 API 初稿，为前端联调铺平了道路。

## 🚩 挑战与阻碍
- 暂无。

## 📅 明日计划
- 开始编写 V1.2 版本的前端逻辑。
- 完善 API 文档的细节。
